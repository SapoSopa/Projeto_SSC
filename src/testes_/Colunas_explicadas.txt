# explicações de cada coluna dos .csv, usando gpt
scp_statements.csv:
Colunas= description,diagnostic,form,rhythm,diagnostic_class,diagnostic_subclass,Statement Category,SCP-ECG Statement Description,AHA code,aECG REFID,CDISC Code,DICOM Code

description
  - Código abreviado da condição detectada no ECG (ex.: NDT, LVH, AFIB).
  - Usado como identificador padronizado de achados clínicos.

diagnostic
  - Descrição sucinta da condição (ex.: "non‑diagnostic T abnormalities", "left ventricular hypertrophy").
  - Facilita a interpretação clínica do traçado.

form
  - Categoria que indica se a declaração refere-se à forma do traçado (ex.: ST‑T, QRS).
  - Representado numericamente, típico valor = 1.0.

rhythm
  - Categoria relacionada ao ritmo cardíaco (ex.: arritmias).
  - Também representado numericamente, típico valor = 1.0.

diagnostic_class
  - Classe ampla da condição, agrupando declarações por tipo geral:
    MI   = Myocardial Infarction
    HYP  = Hipertrofia
    CD   = Conduction Disturbances
    STTC = ST‑T Changes
    NORM = Normal ECG

diagnostic_subclass
  - Subcategoria mais específica dentro da classe acima:
    ex.: IMI (inferior myocardial infarction), LVH (left ventricular hypertrophy), AFIB.
  - Permite distinções detalhadas de condições cardíacas.

Statement Category
  - Categoria textual legível que contextualiza a condição:
    ex.: "Ventricular Hypertrophy", "Myocardial Infarction", "Conduction Disturbances".

SCP‑ECG Statement Description
  - Descrição completa da anormalidade observada no ECG:
    ex.: "Inferior myocardial infarction", "sinus arrhythmia", "non‑specific ST elevation".

AHA code
  - Código padronizado pela American Heart Association para a condição:
    ex.: 161 = inferior MI; 142 = LVH; 50 = AFIB.
  - Útil para interoperabilidade clínica.

aECG REFID
  - Identificador de referência no protocolo aECG (Annotated ECG).
  - Nem sempre preenchido, mas pode ser usado para vinculação com outros padrões clínicos.

CDISC Code
  - Código conforme padrão CDISC (Clinical Data Interchange Standards Consortium).
  - Usado em estudos clínicos e interoperabilidade de dados.

DICOM Code
  - Código conforme padrão DICOM (Digital Imaging and Communications in Medicine).
  - Relevante para integração com sistemas de registros clínicos e imagens médicas.

Explicação das colunas do arquivo CSV (dados cardiovasculares — PTB‑XL)

ecg_id
  - Identificador único de cada ECG (registro).

patient_id
  - Identificador do paciente (um paciente pode ter múltiplos ECGs).

age
  - Idade do paciente no momento do exame (anos).

sex
  - Sexo do paciente (por exemplo: 0 = feminino, 1 = masculino).

height
  - Altura, em centímetros (nem sempre disponível).

weight
  - Peso, em quilogramas (nem sempre disponível).

nurse
  - Código ou identificação da enfermeira que realizou o exame.

site
  - Local ou hospital onde o ECG foi coletado.

device
  - Equipamento utilizado na gravação do ECG (ex.: AT‑6, CS‑12 E).

recording_date
  - Data (e às vezes hora) da gravação do ECG.

report
  - Relatório textual automatizado ou médico descrevendo o ECG (ex.: sinus rhythm, ST‑T changes).

scp_codes
  - Probabilidades de subclasses diagnósticas (ex.: {'IMI':50.0, 'NDT':100.0}) indicando a confiança em cada condição detectada no traçado ECG.

heart_axis
  - Eixo elétrico cardíaco em graus, usado para detectar desvios de eixo (ex: desvio para a esquerda ou direita).

infarction_stadium1
  - Estágio estimado de infarto (por exemplo, Stadium I, II, III), se presente.

infarction_stadium2
  - Segundo estágio (ou estágio complementar) de infarto, se disponível.

validated_by
  - Se o exame foi validado, geralmente por sistema automático ou pipeline institucional.

second_opinion
  - Se houve segunda opinião clínica ou revisão adicional no exame.

initial_autogenerated_report
  - Se o relatório inicial foi gerado automaticamente (True/False).

validated_by_human
  - Indica se o exame e/ou relatório foi validado por um humano (True/False).

baseline_drift
  - Indica presença de deriva de linha de base no sinal, que pode afetar a leitura de componentes lentos do ECG.

static_noise
  - Indica ruído estático contínuo na gravação (interferência).

burst_noise
  - Indica ruído súbito ou transiente (picos falsos).

electrodes_problems
  - Problemas na colocação dos eletrodos (mau contato, circuito aberto etc.).

extra_beats
  - Presença de batidas extra (ectópicas) durante o ECG.

pacemaker
  - Indica se o paciente possui marca‑passo (implante ativo), influenciando características no ECG.

strat_fold
  - Número da partição estratificada usada em validação / treino de modelos de aprendizado de máquina (cross‑validation).

filename_lr
  - Caminho relativo para o arquivo de ECG em baixa resolução (100 Hz).

filename_hr
  - Caminho relativo para o arquivo de ECG em alta resolução (500 Hz).
